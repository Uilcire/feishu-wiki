# 程序搜索与代码合成（Program Search & Code Synthesis）

## 概述

程序搜索是指在可执行程序的空间中进行搜索以找到满足特定目标的代码。与标准 RL 在连续参数空间中优化不同，程序搜索在离散的代码空间中探索 —— 每个候选方案是一段可执行、可检查的程序。LLM 的出现使程序搜索获得了强大的变异算子（Mutation Operator）：LLM 可以基于反馈理解性地重写代码，而非随机变异。

## 核心思想

- **优化对象是代码，不是权重** —— 产出物是透明、可检查、可执行的程序，而非不透明的神经网络参数 [arXiv:2603.03329]
- **LLM 作为变异算子** —— 传统进化算法使用随机变异；LLM 引导的程序搜索使用"理解性变异"，基于错误反馈有针对性地改进代码 [arXiv:2603.03329]
- **树搜索 + 多臂老虎机** —— 维护候选程序的树结构，用汤普森采样（Thompson Sampling）平衡探索与利用 [arXiv:2603.03329]
- **不同方法在"搜索结构"上有显著差异**：AutoHarness 使用结构化树搜索 [arXiv:2603.03329]；Voyager 使用迭代提示修复 [arXiv:2305.16291]；AlphaEvolve 使用进化循环配合自动评估器 [arXiv:2506.13131]；Code as Policies 使用单次生成（无迭代搜索）[arXiv:2209.07753]

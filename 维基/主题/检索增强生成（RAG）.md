# 检索增强生成（RAG, Retrieval-Augmented Generation）

## 概述

检索增强生成是当前 LLM 应用中最常见的知识整合模式。核心流程：将文档切块 → 嵌入向量空间 → 查询时检索相关块 → LLM 基于检索结果生成答案。NotebookLM、ChatGPT 文件上传等产品都采用这个模式。

## 核心思想

- RAG 的核心优势：无需预处理综合，收录成本低，新文档即时可搜索
- RAG 的核心弱点：每次查询从零推导，没有知识积累；跨文档综合需要在查询时实时完成
- 三层 RAG 检索（向量搜索 + 词法搜索 + 图搜索）可以覆盖不同类型的查询需求
- 编译后的维基页面也可以作为 RAG 的索引对象，形成"编译 + 检索"的混合架构
